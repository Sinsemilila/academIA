## Violations détaillées

### V1. Link `text-text-muted` insufficient contrast (FIXED)

- **Element** : `<a class="text-xs text-text-muted hover:text-text-secondary">Mentions légales</a>` (footer Login + autres routes)
- **Contrast** : 2.43:1 (foreground #9ca3af on background #fafafa) — target 4.5:1
- **Cause** : `--color-text-muted` light theme = `oklch(0.714 0.019 261.325)` → trop clair
- **Fix shipped S57** : bump to `oklch(0.530 0.019 261.325)` → contrast ~4.7:1 ✅
- **Side effect** : tous contexts text-muted en light theme (Header XP, KeyboardShortcuts kbd, ProgressionGraph axes, etc.) gagnent en lisibilité. No regression dark theme (token séparé).
- **Validation** : requires frontend rebuild + container restart pour propager (production build, pas dev server)

### V2. Button `bg-teacher` insufficient contrast (PARTIAL FIX — option A applied)

- **Element** : `<button class="bg-teacher text-white">Entrer</button>` (Login submit + autres CTAs)
- **Contrast pre-fix** : 3.67:1 (foreground #ffffff on background #3b82f6) — target 4.5:1
- **Fix shipped S57 (option A teacher only)** : `--color-teacher` oklch(0.623)→oklch(0.55) (≈ #3b82f6 → #2563eb Tailwind blue-600). Contrast ~4.6:1 ✅. Cross-theme uniform (perception-pinned hue+chroma préservée).
- **6 autres accents pending Sinse design review** (similar low contrast risks, deferred pour cohérence design system) :

  | Agent | Current OKLCH L | Brand color | Recommended target | WCAG status |
  |-------|----------------|-------------|---------------------|-------------|
  | maestro | 0.637 | #ef4444 red | oklch(0.54) | ⚠️ probably fails |
  | sensei | 0.627 | #a855f7 purple | oklch(0.53) | ⚠️ probably fails |
  | lehrer | 0.769 | #f59e0b amber | oklch(0.50) | ⚠️ severe (high L) |
  | professore | 0.723 | #22c55e green | oklch(0.50) | ⚠️ severe |
  | pymentor | 0.715 | #06b6d4 cyan | oklch(0.50) | ⚠️ severe |
  | cybermentor | 0.656 | #ec4899 pink | oklch(0.56) | ⚠️ moderate |

- **Decision needed Sinse** : darken les 6 cross-theme similar to teacher (preserves brand identity but uniformly stronger), OR add `bg-{agent}-strong` variants for CTA-only contexts (more tokens, more flexibility), OR audit per-agent visual usage and accept tradeoffs case-by-case.
